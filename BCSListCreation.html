There are six (6) required columns: Experiment, ItemName, Condition, Sentence, Question, and Answer.

Experiment distinguishes between critical items and fillers.

ItemName is to identify an individual item within an experiment. This should be a number. For items this will be the same across all conditions. In the example above there are four rows for item 1, one for each condition in the experiment. You must have as many rows per item as conditions, except for fillers. Each item must have a unique number.

Condition is to identify which condition the item is in within the experiment. Condition names should not contain any spaces or special symbols, except for "." like in the example above. The condition column needs to follow a specific format. Each manipulation needs to be separated by a ".", and each condition within a manipulation needs a unique label.

You can name your conditions what you like but, it is useful to use descriptive names. In the above 2x2 design, the first manipulation is whether the sentence is or is not temporarily ambiguous (the first 2 of the 2x2). For clarity the labels are Ambiguous and Unambiguous. The second maniuplation is whether the ambiguity involves the main verb or relative clause (the second 2 of the 2x2). Again, for clarity the labels are MainVerb and RelativeClause. Filler items all have the same condition and it is marked as "-".

We will be randomizing the lists for you so the order you put items or fillers in does not matter. But remember that we are using a latin square design so participants will only see each item in one of its possible conditions (e.g. the first sentence of item 1, the second sentence of item 2, etc.). Participants will see all the fillers.

Sentence is the stimulus item that will be self-pace read. Within an item, you should use minimally different sentences. In the example above you can see that the differences between Ambiguous and Unambiguous stimuli is one word. The difference between the MainVerb and RelativeClause stimuli is larger but as much of the sentence is kept as similar as possible. Avoid characters other than "," and "." in your stimuli.

The experimentally important regions of the sentences need to be marked using a specific syntax. For readability of the stimuli, the above table does not have this marking but we can reproduce the marking that would go on the first four stimuli here:

The excited fans {moved}@Verb through {the crowd}@PreDisambiguation {and}@Disambiguation {sauntered to}@PostDisambiguation front stage.
The excited fans {moved}@Verb through {the crowd}@PreDisambiguation {broke}@Disambiguation {apart from}@PostDisambiguation Jane accidentally.
The excited fans {flew}@Verb through {the crowd}@PreDisambiguation {and}@Disambiguation {sauntered to}@PostDisambiguation front stage.
The excited fans that {moved}@Verb through {the crowd}@PreDisambiguation {broke}@Disambiguation {apart from}@PostDisambiguation Jane accidentally.

In the example experiment above, the interest is in how reading times are affected at the point of disambiguation. So the main region to mark is the disambiguating word. We mark that word by surrounding it with curly braces and labeling the region (e.g. {broke}@Disambiguation). This marking is not included in what the participants see, it is just to help mark up the data for analysis. The region label can only be upper case and lower case letters (e.g. no "."). You can put more than one word in a region. All items should have the same regions and in the same order.

In the example above there are multiple regions marked. Because effects on reading can "spill over" into later words, it is important to mark the words after the disambiguation point as well (e.g. {apart from}@PostDisambiguation). In the example above, there is a manipulation of parts of the sentence before the disambiguation point, specifically the verb or the sentence complementizer "that". Because of these changes, it is important to also mark those regions. Lastly, it may be important to know if there are changes in reading before the point of disambiguation, so that region should also be marked. How many and which regions are marked depend on your experimental design, think carefully about it. Generally, marking more regions is better than marking fewer.

Question is the comprehension question to ensure that subjects are actually paying attention and not just holding down the space bar. Within an item, you want the same question, or a minimally different question and the questions should lead to the same answer. For critical items, you often want to probe a specific part of the stimulus to make sure that that part was indeed processed correctly. Across all stimuli, however, it is advisable to probe different parts of the sentences to roughly equal extents. For example, if you only asked questions about the first word of all sentences, participants might pick up on that and stop paying attention to the remainder of sentences.

Answer is the correct answer for the comprehension question. Within an item, correct answers should be the same. In the example above all the answers for item 1 is N. Across the experiment you should make sure to have equally many Y and N answers. This will reduce any bias and increase the likelihood of participants paying attention.

To make sure this happens, make half the filler items have a Y answer and the other half have a N answer. Then make sure half your critical items have a Y answer and the other half have an N answer. This works because participants will see all fillers and only 1 of the conditions of critical item. So ensuring each type of item has a 50/50 split will ensure each participant sees a 50/50 split.

Important points about the format

Condition names should only be upper and lowercase letters and periods (see example above). Do not include spaces, numbers or other characters.

Remove blank/empty rows between stimuli.

Every stimulus needs a question and answer.

The sentences and questions will be displayed as-is (once region marking is removed) so do not include extra characters unless you want those characters to be displayed. For example, if your design uses a comma to disambiguate an ambiguous sentence make sure you include commas and be careful with stray spaces (e.g. "this, that" and "this , that" will be displayed differently).

Regions should be indicated with an {words}@Name, e.g.

  • {word}@regionname
  • {several words here}@anotherregionname
  • {This sentence}@subject {has}@verb {multiple regions}@object

Every item (except fillers) should have the same regions and in the same order. If it is necessary, the same region can have a different number of words within an item or across items.

Entering Items

  • Enter each of your items (make sure each item has a different ItemName and the same number of conditions, one per row)
  • Enter your fillers on the same sheet
  • Make sure that within an item questions are similar and the answers are the same.
  • Make sure you have an even number of Y and N question answers.
  • Make sure all items have proper condition names that are sensible.
  • Make sure all items (except fillers) have marked regions.
  • n.b. the program that generates the lists for the applet based on your Excel sheet will ignore any column other than the ones described above, so feel free to have some extra columns if they help you when making your stimuli, but also keep in mind that if you screw up the spelling of your required columns they will be ignored too.
  • Save the file and send it to us.
